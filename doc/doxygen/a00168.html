This header introduces two kinds of assertion macros. 1.: PNAPI_ASSERT(expr) should be used like cassert, to ensure internal behaviour. But instead of terminating the whole program, this macro will cause an exception. 2.: C assertions can be disabled by defining the symbol NDEBUG, so does PNAPI_ASSERT. But some errors are caused by the user (i.e. defining a node name twice or deleting nonexisting nodes). These kind of errors should still be reportet to the customer tool by throwing an exception, so PNAPI_ASSERT_USER(expr, msg) can not be disabled by defining NDEBUG. When "expr" evaluates to false, an exception containing the message "msg" will be thrown.

Both makros can be disabled by defining the symbol PNAPI_USE_C_ASSERTS. They will behave like assert(expr) then.
