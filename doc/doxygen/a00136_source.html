1609  char const *yyarg[YYERROR_VERBOSE_ARGS_MAXIMUM];

1612  - If this state is a consistent state with a default action, then
1613  the only way this function was invoked is if the default action
1614  is an error action. In that case, don't check for expected
1615  tokens because there are none.
1616  - The only way there can be no lookahead present (in yytoken) is
1617  if this state is a consistent state with a default action.
1618  Thus, detecting the absence of a lookahead is sufficient to
1619  determine that there is no unexpected or expected token to
1620  report. In that case, just report a simple "syntax error".

1622  a consistent state with a default action. There might have
1623  been a previous inconsistent state, consistent state with a
1624  non-default action, or user semantic action that manipulated
1625  yychar.

1627  correct lookahead information, and it depends on the parser not
1628  to perform extra reductions after fetching a lookahead from the
1629  scanner and before detecting a syntax error. Thus, state
1630  merging (from LALR or IELR) and default reductions corrupt the

1632  canonical LR with one exception: it will still contain any
1633  token that will not be accepted due to an error action in a

1656  break;
1657  }

1659  yyarg[yycount++] = yytname_[yyx];
